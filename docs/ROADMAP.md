# CppCmakeProjectTemplate — Roadmap & Ideas

For completed capabilities, see [CAPABILITIES.md](CAPABILITIES.md).

---

## 🔜 Backlog

### Performance

- **Hot Reloading:** C++ hot-reloading (LLVM JIT / cr.h) requires significant runtime scaffolding and OS-specific shared library reload. ccache + unity builds already minimize rebuild latency.

---

## 💡 Ideas — New Capabilities

- **Monorepo Support:** Multiple independent `tool.toml` files in subdirectories, with a root-level orchestrator (`tool workspace`).
- **Plugin System v2:** Community plugins installable via `tool plugin install <name>` from a plugin registry.

## 💡 Ideas — Developer Experience

- **Interactive TUI v2:** Full-screen TUI mode with menu navigation, build status dashboard, log viewer.
- **Reproducible Build Verification:** Bitwise reproducibility check — build twice, compare hashes.

## 💡 Ideas — Radical

- **Self-Hosting:** The generator generates its own `scripts/` infrastructure — full bootstrap from a minimal seed.
- **Language Server Protocol (LSP) for tool.toml:** IDE support with completion, validation, and hover docs for `tool.toml` editing.
- **AI-Assisted Code Generation:** Integrate LLM APIs for generating domain-specific C++ code based on `tool.toml` metadata and library descriptions.
- **Multi-Language Scaffolding:** Extend the generator to scaffold Rust/Go/Python companion libraries alongside C++, with FFI bindings auto-generated.

---

## 📜 Governance

- **SemVer:** Semantic versioning for CLI and templates.
- **LTS Support:** Maintain both Latest and LTS release streams.

---

## 💡 Strategic Recommendations

1. **Keep Template Logic Minimal:** Keep template files small; implement complex logic in Python.
2. **Atomic Operations:** Ensure file mutations support rollback.
3. **Static Analysis Integration:** Integrate `clang-tidy --fix` into the `check` command to improve quality.

## Long-Term Vision

The entire C++ project is generated by scripts from a single `tool.toml` configuration. Only `scripts/` and `tool.toml` are hand-maintained — all CMakeLists.txt, C++, CI, Docker, IDE configs, and documentation are generated.

### Final Goal: Cargo + Cookiecutter + Bazel + IaC Pattern

```
tool.toml (declare) → tool generate (produce) → tool build (compile) → tool release (ship)
```

The scripts manage the full lifecycle: scaffolding, building, testing, packaging, and deployment.
